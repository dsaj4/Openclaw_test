## 🚀 使用指南

### 第一步：初始化工作空间

```bash
# 创建工作空间目录结构
mkdir -p /workspace/openclaw_research_test/{skills,scripts,outputs/{reports,logs,data},temp,docs}
```

### 第二步：启动测试主Agent

使用总提示词启动测试主Agent：

```
请作为OpenClaw投研测试主Agent，启动测试框架。

测试场景: [选择案例，如F1]
案例编号: [编号]
主要Skill: [skill名称]
难度等级: [低/中/高/极高]
```

### 第三步：执行测试

主Agent将按照以下流程执行：

1. **初始化监控** - 启动token/时间/交互记录器
2. **Skill准备** - 检查并安装所需skill
3. **执行测试** - 按照案例提示词完成任务
4. **收集结果** - 停止监控，收集性能指标
5. **生成报告** - 生成结构化测试报告

### 第四步：查看结果

测试结果将保存至：
- 测试报告: `/workspace/openclaw_research_test/outputs/reports/`
- 执行日志: `/workspace/openclaw_research_test/outputs/logs/`
- 输出数据: `/workspace/openclaw_research_test/outputs/data/`

## ⚠️ 注意事项

1. **数据安全**: 所有测试数据保存在本地工作空间，不上传云端
2. **API密钥**: 妥善保管金融数据API的账号密码
3. **资源消耗**: 极高难度案例可能需要大量计算资源
4. **网络连接**: 确保网络连接稳定，特别是数据API调用
5. **测试环境**: 建议在测试环境执行，避免影响生产数据
