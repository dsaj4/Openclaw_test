# OpenClaw 中难度案例提示词集

本文件仅保留当前有效的中难度案例，并优先给出可直接执行的 F16 细化测试提示词。

## 当前有效案例

- F16: 股票分析 Skills 个股深度分析
- G2: 迭代投研框架
- GF1: PDF 财报读取 + Word 报告生成
- GF2: PDF 材料自动生成路演 PPT
- GF3: 金融数据接入 Stock Watcher
- GF7: 技术分析与 K 线图分析

## F16 细化测试目标

验证 `stock-analysis` skill 是否真的调用其 `SKILL.md` 中声明的 Yahoo API，而不是只写一份像样的股票分析 Markdown。

### F16 强制要求

1. 必须调用以下 API：
   - `Yahoo/get_stock_profile`
   - `Yahoo/get_stock_insights`
   - `Yahoo/get_stock_chart`
   - `Yahoo/get_stock_holders` 或 `Yahoo/get_stock_sec_filing` 至少一个
2. 必须生成两份产物：
   - 分析报告：`outputs/data/F16_stock_analysis_report.md`
   - 原始 API 结果：`outputs/data/F16_api_results.json`
3. 原始 API 结果必须单独保存为 `.json` 文件，不能写进 Markdown 里冒充完成。
4. 如果未调用真实 API，即使报告写得完整，也视为测试失败。

## F16 细化提示词

```text
【测试案例 F16】股票分析 Skills 个股深度分析

你必须使用 stock-analysis skill 完成对单个股票的真实分析，不允许只生成一份通用 Markdown 报告来冒充执行结果。

【测试目标】
验证 stock-analysis skill 是否会按其 SKILL.md 定义调用真实 Yahoo API，并输出：
1. 一份分析报告
2. 一份原始 API 结果文件

【分析标的】
- symbol: AAPL
- region: US
- lang: en-US

【必须调用的 API】
1. Yahoo/get_stock_profile
2. Yahoo/get_stock_insights
3. Yahoo/get_stock_chart
4. Yahoo/get_stock_holders 或 Yahoo/get_stock_sec_filing 至少一个

【执行步骤】
Step 1 - 环境检查
1. 确认 stock-analysis skill 已安装
2. 确认其配置已完成
3. 确认 Yahoo 数据接口可调用

Step 2 - 真实 API 调用
1. 调用 Yahoo/get_stock_profile 获取公司信息
2. 调用 Yahoo/get_stock_insights 获取技术面、估值、评级与研究摘要
3. 调用 Yahoo/get_stock_chart 获取价格走势数据
   - 建议参数：interval=1d, range=6mo
4. 再调用以下其中一个：
   - Yahoo/get_stock_holders
   - Yahoo/get_stock_sec_filing

Step 3 - 保存原始 API 结果
1. 将所有 API 返回结果汇总为一个结构化 JSON
2. 保存到：outputs/data/F16_api_results.json
3. 不允许仅在 Markdown 中粘贴摘要来替代这个 JSON 文件

Step 4 - 生成分析报告
报告必须包含：
1. 公司概览
2. 技术与走势观察
3. 估值与评级信息
4. 持股/监管文件补充观察
5. 综合结论与风险提示

将报告保存到：
outputs/data/F16_stock_analysis_report.md

【通过标准】
- 四类要求中的 API 已真实调用
- JSON 文件存在且包含原始 API 结果
- Markdown 报告存在且引用 API 结果形成结论

【失败标准】
- 未调用必需 API
- 只有 Markdown 报告，没有 JSON 结果文件
- JSON 文件不是原始 API 结果，而是人工摘要
- 用通用股票常识替代 skill 执行
```
