# F14 测试规范

## 测试名称

F14 PB-ROE 选股策略构建

## 测试目标

验证 OpenClaw 能否完成以下完整链路：

1. 使用 Chrome 浏览器打开知乎并检索 PB-ROE 选股策略
2. 总结一份 PB-ROE 策略构建框架
3. 使用同花顺 API 真实提取策略所需数据
4. 构建经典 PB-ROE 选股策略
5. 计算最新一期持仓组合
6. 输出策略报告和最新持仓结果文件

## 强制要求

1. 必须使用浏览器完成知乎检索，不允许直接凭记忆写 PB-ROE 框架。
2. 必须使用同花顺 API 真实取数，不允许手工伪造指标。
3. 必须单独输出持仓结果文件，不允许只在 Markdown 里贴一个表格冒充完成。
4. 报告和持仓文件必须同时存在。

## 必要依赖

- `agent-browser`
- `10jqka-api`
- `strategy-builder`

## 配置要求

至少需要确认：

- Chrome 可用
- 同花顺 API 账号配置完成
- 同花顺 SDK 或 HTTP 接口可用
- A 股股票池和关键因子字段可提取

## 强制执行步骤

### Step 1：浏览器检索策略框架

1. 使用 Chrome 打开知乎
2. 搜索关键词：`PB-ROE 选股策略`
3. 阅读至少 1 篇高相关内容
4. 提炼策略框架，至少包括：
   - 策略核心原理
   - 主要指标定义
   - 选股条件
   - 评分方法

### Step 2：定义经典 PB-ROE 策略

建议至少包含：

- 股票池：沪深 A 股
- 剔除 ST
- PB 条件：PB < 3
- ROE 条件：ROE > 10%
- 评分思路：
  - 低 PB 更优
  - 高 ROE 更优
  - 综合得分排序

### Step 3：同花顺 API 真实取数

必须从同花顺 API 提取构建策略所需的数据，至少覆盖：

- 股票代码
- 股票名称
- PB
- ROE

如有能力，可补充：

- 行业
- 市值
- 最近财报期

### Step 4：计算最新一期持仓组合

1. 对股票池执行筛选
2. 计算综合得分
3. 取最新一期 Top 20 持仓

保存到：

- `outputs/data/F14_latest_portfolio.csv`

CSV 至少包含：

- 排名
- 股票代码
- 股票名称
- PB
- ROE
- 综合得分

### Step 5：生成策略报告

报告必须包含：

1. 知乎检索得到的策略框架摘要
2. 最终采用的 PB-ROE 规则
3. 同花顺 API 数据字段说明
4. 评分方法
5. 最新持仓组合摘要
6. 风险与局限性说明

保存到：

- `outputs/data/F14_pb_roe_strategy_report.md`

## 通过标准

1. 已使用浏览器完成知乎检索
2. 已使用同花顺 API 真实取数
3. 持仓 CSV 文件存在
4. 策略报告存在
5. 报告与 CSV 内容一致

## 失败标准

1. 未进行浏览器检索
2. 未调用同花顺 API
3. 只有 Markdown 报告，没有 CSV 持仓文件
4. 持仓表为人工编造，不来自真实数据
