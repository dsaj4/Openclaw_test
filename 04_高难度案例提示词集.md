## G5 细化测试目标

验证 OpenClaw 能否为阿里巴巴建立一个真正可运行的自动化监控任务，而不是只生成一份一次性的报告草稿。

## G5 强制要求

1. 必须建立一个在 2026-03-11 18:18（Asia/Shanghai）执行的一次性测试调度。
2. 必须抓取 BABA 或 09988 的最新收盘价、成交量和涨跌幅。
3. 涨跌幅绝对值大于 3% 时，必须在报告开头给出异动警报。
4. 必须用联网浏览器在过去 24 小时内检索阿里云、通义千问、AI Agent 三个方向的资讯。
5. 必须只保留 3 条最有战略价值的资讯，并给出原始链接与 50 字以内洞察。
6. 必须输出 Markdown 早报，并真实调用 Email 插件发送。
7. 必须保留结构化数据文件和发送日志。

## G5 细化提示词

```text
【测试案例 G5】阿里巴巴专属自动化监控早报

请作为一个专业的科技行业研究助理，为阿里巴巴建立一个专属自动化监控任务。
你必须真正创建自动化、真正抓取数据、真正检索新闻、真正发送邮件，不允许只生成一份 Markdown 草稿来冒充完成。

【调度要求】
- 测试模式
- 一次性执行
- 触发时间：2026-03-11 18:18（Asia/Shanghai）

【步骤 1：盘面数据监控】
1. 调用金融数据接口，抓取 BABA 或 09988 的：
   - 最新收盘价
   - 成交量
   - 涨跌幅
2. 将结果保存为：
   outputs/data/G5_market_snapshot.json
3. 如果涨跌幅绝对值超过 3%，则在最终早报开头加入显著异动警报。

【步骤 2：定向网络搜索】
使用联网浏览器，严格检索过去 24 小时内关于以下主题的新闻：
1. 阿里云 / Alibaba Cloud
2. 通义千问
3. AI Agent / 智能体应用

要求：
- 排除自媒体八卦
- 排除重复通稿
- 仅保留 3 条最有战略价值的资讯
- 每条资讯必须包含：
  - 标题
  - 原始链接
  - 50 字以内技术/商业洞察

将结果保存为：
outputs/data/G5_news_digest.json

【步骤 3：生成极简早报】
生成 Markdown 格式报告：
- 文件路径：
  outputs/data/G5_aliyun_morning_brief.md
- 邮件标题必须严格为：
  【阿里早报】YYYY-MM-DD：BABA盘面追踪与云/AI生态动态

【步骤 4：邮件发送】
1. 使用 Email 插件将最终早报发送至运行时配置中的目标邮箱
2. 保存发送日志：
   outputs/data/G5_email_send_log.json
3. 发送成功后，在当前对话回复：
   老板，今日阿里专属早报已发送。

【通过标准】
- 已创建 2026-03-11 18:18（Asia/Shanghai）一次性调度
- 盘面 JSON 存在
- 新闻 JSON 存在
- Markdown 早报存在
- 邮件发送日志存在且成功

【失败标准】
- 没有创建调度
- 没有发送邮件
- 只有 Markdown 没有结构化数据文件
- 没有原始链接
- 保留了低价值八卦或重复稿件
```
